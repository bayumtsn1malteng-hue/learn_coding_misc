SERBA-SERBI LINUX

Cara Mount Drive Secara Otomatis

1. Check system mount directory

cd /mnt/

2. Locate Unknown Drive Path

lsblk

3. Create Mount Folder

sudo mkdir /mnt/mydrive

mydrive bisa diganti dengan nama lain

4. Cari tahu UUID partisi

sudo blkid /dev/sda1

Ganti /dev/sda1 dengan partisi yang diinginkan. Catat UUID dan tipe (ntfs, etc)

Cadangkan konfigurasi penting

sudo cp /etc/fstab /ets/fstab.bak

Edit file fstab

sudo nano /etc/fstab

Tambahkan di paling bawah baris berikut:

UUID=D86094D86094BEA6    /mnt/mydrive    ntfs-3g    defaults,windows_names,uid=1000,gid=1000    0    2

Arti kode

  • ntfs-3g: Ini adalah driver terbaik di Ubuntu agar Anda bisa membaca (read) sekaligus menulis/mengedit (write) file di dalam drive NTFS milik Windows tersebut.
  • uid=1000,gid=1000: Ini memastikan bahwa akun Ubuntu Anda memiliki hak penuh untuk membuat, menghapus, atau mengedit file di drive tersebut tanpa perlu mengetik sudo terus-menerus.
  • windows_names: Mencegah Anda membuat file dengan karakter yang dilarang oleh Windows (seperti tanda tanya ? atau titik dua :), sehingga drive tetap aman jika nanti dibuka lagi di Windows.
  • Ganti UUID dan /mnt/drive sesuai dengan yang diinginkan
sudo mount -a

Kalau tidak ada error berarti aman. Bila muncul:

tfs-3g-mount: failed to access mountpoint /mnt/mydrive: No such file or directory
mount: (hint) your fstab has been modified, but systemd still uses
       the old version; use 'systemctl daemon-reload' to reload.
buat ulang folder mountpoint sudo mkdir -p /mnt/drive segarkan sistem ubuntu sudo systemctl daemon-reload jalankan kembali sudo mount -a

Mitigasi

Kalau error, buka kembali sudo nano /etc/fstab atau kembalikan file asli dengan sudo cp /etc/fstab.bak /etc/fstab
